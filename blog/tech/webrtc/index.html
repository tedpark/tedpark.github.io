WebRTC + SFU + React Native get hands dirty!

WebRTC 프로토콜을 알게 된건 와이프님의 “실시간 요리 방송 서비스를 만들어줘” 에서 출발했다.

  1. 처음엔 가볍게 Socket.io 에 node.js 붙이고 스마트폰 디바이스에서 동영상 압축해서 서버로 보낼 생각이였다.

  2. 하지만 나의 기대와는 다르게 3 초 이상씩 동영상이 딜레이 되었다.

  3. 몇달 후, WebRTC 를 알게 되었고, 미디어 서버 없이 1:1 은 큰 문제 없이 구현 되었다. (딜레이 1 초 미만)

  4. WebRTC 를 사용하면서 STUN 과 TURN 에 대해 구별하고 이해하고 구현 할 수 있게 되었다.

  5. 다시 시작된 와이프의 요청 사항 “여러명이서 동시에 할 수는 없나?”

  6. 다시 시작된 digging 중 SFU 와 MCU 에 대해 알게 되었고, 미디어 서버에 관한 힌트를 얻을 수 있었다.

최종 구현은 아래와 같다.

iOS , Android , Web 에서 모두 동작하는 실시간 쌍방향 비디오 패킷을,
udp 방식으로 쏘고 코덱은 가변적인 vp9 스마트폰은 어쩔 수 없이 h.264 를 쓰도록 구현했다.

프로토콜은 WebRTC. (당연하지만)

미디어 서버는 홍콩이나 싱가폴 어딘가에 있을 Azure Cloud Server 에, Docker 로 scalability 하게 구현했다.
모든 부하는 미디어 서버가 담당하고, 클라이언트는 그저 자신의 미디어만 미디어서버로 전송한다.
모바일 기기의 부족한 전송 대역과 인코딩 디코딩 부하를 서버로 보내고 나서야 여러명이서,
비디오 채팅이 가능하게 되었다.

현재 미디어 서버는 모든 패킷을 녹화하고 저장한다.
각 room 은 쌍방향은 100 명이상,
단순 미디어 스트리밍은 10000 명 이상이지만 이론상 무한대이다.
(마치 무한한 지랫대가 있다면 달도 들 수 있다와 유사한 발언이다.)
처리하도록 설계되었다.

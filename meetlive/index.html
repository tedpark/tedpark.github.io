MeetLive

온라인으로 만나는 리얼타임 밋업 플랫폼
지역에 상관없이 리얼타임 밋업이 가능한 플랫폼이다.
React Native 로 만들었으며, WebRTC + TURN + Media Server 를 사용,
실시간으로 약 100명 정도 동시에 화상회의가 가능하다.
WebRTC 라는 공통의 프로토콜을 사용하여 미디어 데이터를 주고 받는다.
iOS, Android, Web 모두 사용가능한 멀티 플랫폼 서비스다.
현재 한국에서 하나밖에 없는 리얼타임 밋업 플랫폼 플랫폼이다.

1. WebRTC 를 알게 된건 와이프님의 “실시간 요리 방송 서비스를 만들어줘” 에서 출발했다.

2. 처음엔 가볍게 Socket.io 에 node.js 붙이고 스마트폰 디바이스에서 동영상 압축해서 서버로 보낼 생각이였다.

3. 하지만 나의 기대와는 다르게 3초 이상씩 동영상이 딜레이 되었다.

4. 그리고 WebRTC 를 알게 되었고, 서버 없이 1:1 은 큰 문제 없이 구현 되었다.

5. WebRTC 를 구현하면서 STUN 과 TURN 에 대해 구별하고 이해하고 구현 할 수 있었다.

6. 다시 시작된 와이프의 요청 사항 “여러명이서 할 수는 없나?”

7. 다시 시작된 digging 중 SFU 와 MCU 에 대해 알게 되었고, 미디어 서버에 관한 힌트를 얻을 수 있었다.


==================== 최종 구현은 아래와 같다. =======================


iOS , Android , Web 에서 모두 동작하는 실시간 쌍방향 비디오 패킷을 udp 방식으로 쏘고,

코덱은 가변적인 vp9 스마트폰은 h.264를 쓰도록 구현했다.

프로토콜은 WebRTC.

미디어 서버는 홍콩이나 싱가폴에 있는 Azure 에 Docker 로 scalability 하게 구현했다.

모든 부하는 미디어 서버가 담당하고, 클라이언트는 그저 자신의 미디어만 미디어서버로 전송한다.

미디어 서버는 모든 패킷을 녹화하고 저장한다.

각 room 은 쌍방향은 100명이상,

단순 미디어 스트리밍은 10000명이상.

처리하도록 설계되었다.
